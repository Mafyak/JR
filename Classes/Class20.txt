Web Java EE

Веб сервер - сервер принимающий и выдающий HTTP запросы.
Сервер приложений - фреймворк, для эффективного исполнения программ.
Контейнер сервлетов - сервер который поддерживает работу сервлетов

HTTP
  Методы:
  GET - запрос документа. Все параметры передаются в урле
  HEAD - выдаётся только заголовок документа, сам документ не передаёт.
  POST - применяется для передачи данных на сервер.
  PUT - для обновления, размещения документа на сервере.
  
  Путь/версия
  
  Запросы:
   Connection
   User-Agent
   Accept
   Referer
   Host
   Accept-Language
  
 Ответ
  JAR - класы xml 
  WAR - jar Для сервлетов jsp html, для запуска томкат предложения
  EAR - jar 
  
Базовая структура веб-приложения включает корневую директорию, WEB-INF директ и дескриптор развёртывания web.xml
web.xml используется для Servlet и JSP.

Сервлет - компонент на стороне ссервера, обрабатывает запросы и возвращает ответы динамически
Цикл - загрузка класса, создание экземпляра, init, service, destroy
servletrequest\servletresponse - Интерфейсы, определённые в пакете servlet
  
  Servlet 3.0 позваляет не использовать web.xml а использовать аннотации
  
  Веб приложение поставляется в виде архива .war, содержащего все его файлы.
  WEB-INF содержит 2 поддиректории
    /classes - для .class файлов сервлетов
    /lib - для библиотек
    web.xml находится непосредственно в WEB-INF
    
  Структура Tomcat
    /bin - файлы запуска
    /common - библиотеки
    /conf -  конфига
    /logs - логи
    /lib - библиотеки
    /webapps - приложение(.war файл)
    
  Forward(), include(), sendRedirect()
  Forward - редиректит, на стороне сервера, внутри сервера\ресурса(урл не меняется), быстрее  
  include - включает содержимое ресурса
  sendRedirect - редирект, на стороне клиента, внутри и вне сервера\ресурса, два вызова, защита от F5.
